命名（命名空间）

不好的例子

.id.red.top.img.div

较好的例子

.content.navigator.header.sidebar.logo

更好的例子

.show_content.show_nav.show_hd.show_sidebar.show_logo

可重复使用

可重用意味着模块化，批量生产

.mod_basic.mod_primary.gb_nav.gb_sidebar

可维护

可维护的前提是，模块化和标准化

可扩展

举个栗子

栗子

原来是这样子的：

.sidebar_a {
		background:#ddd;
		width:100px;
		blablabla.....
}

然后小G这样改：

.sidebar_a {
		background:#aaa;
		width:200px;
		blablabla.....
}

很明显，这样是很不利扩展的，正确的做法是：

.index .sidebar_a {
		background:#ddd;
		width:100px;
}

又或者给sidebar_a加个类名“.index_sidebar_a”

.index_sidebar_a {
		background:#aaa;
		width:200px;
}

记住：软件实体（类，模块，函数等）应对扩展开放，对修改闭合。
